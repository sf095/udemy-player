# Spec: Filter Summary TXT Files from Resources & Keep Tabs Visible in Theater Mode

## Objective

Fix two bugs in the Udemy Offline Player:

1. **Summary `.txt` files incorrectly listed as companion resources** — The backend scanner treats every non-video, non-subtitle file as a companion resource. Summary `.txt` files (generated by the app's own summarization feature at `backend/server.js:773`, following the pattern `<basename>.summary.<langcode>.txt`) appear in the sidebar resource badge count and the resource list, cluttering the UI with non-actionable items. Other `.txt` files that are legitimate resources should still be listed.

2. **Theater mode blocks access to companion resources** — When theater mode is active, the `.stage-tabs` element (containing "Video Lesson" / "Companion Resources" tabs) is hidden via CSS. Since clicking the "Companion Resources" tab already correctly exits theater mode (line 876 of `App.jsx`), the only problem is that the tab is invisible and unclickable.

## Boundaries

- **Always do:** Preserve existing resource type detection (pdf, html, quiz, url, file). Keep the existing behavior of exiting theater mode when switching to companion resources.
- **Ask first:** Adding new resource types or changing the theater mode UX beyond making tabs visible.
- **Never do:** Delete or alter `.txt` files on disk. Break the existing resource filtering for other file types.

## Implementation Plan

### Fix 1: Filter summary `.txt` from resources (1 file, 1 line)

In `backend/scanner.js`, in the resource detection block (the catch-all `else if` around line 125), add a condition to skip `.txt` files that match the summary naming pattern `<basename>.summary.<langcode>.txt`:

```
// Before (approx line 125):
} else if (file.name.toLowerCase() !== '.ds_store') {

// After:
} else if (!(file.ext === '.txt' && file.name.includes('.summary.')) && file.name.toLowerCase() !== '.ds_store') {
```

This targets only summary files generated by the app's `/api/summarize-lesson` endpoint (server.js:773), while leaving other `.txt` files intact as resources.

### Fix 2: Keep tabs visible in theater mode (1 file, remove 1 CSS rule)

In `frontend/src/index.css`, remove the CSS rule that hides `.stage-tabs` in theater mode (lines 1204-1212):

```
/* Remove this block: */
.app-container.theater-mode .stage-tabs {
  height: 0;
  padding-top: 0;
  padding-bottom: 0;
  overflow: hidden;
  border-bottom-width: 0;
  opacity: 0;
  pointer-events: none;
}
```

The header and stage banner hiding in theater mode remain unchanged — only the tabs become visible.
