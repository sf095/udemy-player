# Spec: Automatically Create Timeline and Summary Settings

## Objective
Enable users to toggle whether video timeline chapters (timeline segments) and/or lesson summaries are automatically generated using AI when a video lesson is loaded. Currently, these must be generated by clicking manual buttons. Providing toggleable automatic generation enhances the user experience by automating the generation process for students who want a hands-free, fully enriched layout upon clicking any lesson.

### User Stories
1. **Automated Timeline Chapters**: As a student, I want the player to automatically generate the timeline segments using the transcript if they don't already exist, so I can see video chapters immediately.
2. **Automated Lesson Summary**: As a student, I want the player to automatically generate a lesson summary in my preferred language if it does not exist, so I can read it immediately.
3. **Toggle Controls**: As a student, I want to be able to enable or disable auto-creation of both timeline chapters and summaries independently in the Application Settings.
4. **Graceful Failures**: As a student, if my API Key is not set or the generation fails, I want the application to handle it gracefully without disruptive errors.

## Assumptions
1. Both `autoCreateTimeline` and `autoCreateSummary` will default to `false` in `DEFAULT_SETTINGS` to prevent unintended AI API usage and billing.
2. Auto-generation requires a primary subtitle file to be available for the lesson. If no subtitles are available, auto-generation will be skipped.
3. Auto-generation will only run if the active AI Provider key is configured. If not configured, auto-generation is skipped.
4. Automatic timeline chapter generation will use the configured timeline chapter language (`autoCreateTimelineLang`, defaulting to 'English').
5. Automatic summary generation will use the configured summary language (`autoCreateSummaryLang`, defaulting to 'en').

## Tech Stack
- **Frontend**: React 19, Vite, Lucide Icons, Vanilla CSS
- **Backend/Desktop**: Node.js, Express, Local JSON DB (`progress_db.json`)

## Commands
- **Dev (Concurrent)**: `npm run dev`
- **Frontend Dev**: `npm run dev --prefix frontend`
- **Backend Dev**: `npm run dev --prefix backend`
- **Production Build (Frontend)**: `npm run build --prefix frontend`

## Project Structure
- `frontend/src/App.jsx` — Manages default settings state and handles updating settings. Passes settings to components.
- `frontend/src/components/SettingsModal.jsx` — Provides UI checkboxes/toggles for `autoCreateTimeline` and `autoCreateSummary`, along with dropdowns for `autoCreateTimelineLang` and `autoCreateSummaryLang`.
- `frontend/src/components/VideoPlayer.jsx` — Houses the video playback timeline. Automatically triggers chapter generation in the configured language if `autoCreateTimeline` is true and chapters don't exist yet.
- `frontend/src/components/NotesPanel.jsx` — Houses the lesson summary. Automatically triggers summary generation in the configured language if `autoCreateSummary` is true and summary isn't cached yet.
- `backend/server.js` — Stores default settings, backfills missing fields, and handles settings persistence to `progress_db.json`.

## Code Style
- **React**: Functional components, React Hooks (`useEffect`, `useState`, `useCallback`).
- **Styling**: Vanilla CSS, using matching CSS variables (e.g. `var(--primary)`, `var(--text-secondary)`) and existing layout classes.
- **Snippet example**:
```javascript
// Check setting in VideoPlayer.jsx to trigger generation
if (!data.exists && autoCreateTimeline && subtitlePath && hasApiKey && !loadingChapters) {
  handleGenerateChapters(autoCreateTimelineLang || selectedChapterLang);
}
```

## Testing Strategy
- **Manual Verification**:
  1. Open Application Settings (using Settings icon or `,` shortcut). Verify the new toggles for "Automatically Create Timeline" and "Automatically Create Summary" are visible, along with language dropdown selectors.
  2. Toggle both settings ON, select target languages (e.g. 'Vietnamese' for timeline, 'vi' for summary), and click "Save Settings". Check that `progress_db.json` contains:
     - `autoCreateTimeline: true`
     - `autoCreateTimelineLang: "Vietnamese"`
     - `autoCreateSummary: true`
     - `autoCreateSummaryLang: "vi"`
  3. Load a lesson that does not have chapters/summary generated yet. Verify that:
     - Chapters are automatically generated in Vietnamese and segments appear on the timeline.
     - Summary is automatically generated in Vietnamese and displays in the summary tab.
  4. Disable the settings, load a new lesson without chapters/summary. Verify that they are NOT auto-generated, but the "Generate" buttons are shown.
  5. Delete/clear AI API keys in settings. Load a lesson with settings enabled. Verify that the app handles the lack of keys gracefully (e.g. no infinite generation loops or crash loops).

## Boundaries
- **Always**: Keep the manual "Generate" and "Re-generate" controls fully functional even when auto-creation is enabled.
- **Ask first**: N/A
- **Never**: Spam notifications or popups for background auto-generation errors. Show the status inline (e.g., standard error text or keeping manual generation button with error label).
