# Implementation Plan: HTML Lesson Anchor Link Handling

## 1. Major Components & Dependencies
- **DocViewer.jsx**: The only React component that needs modification.
- **Vite Proxy**: Reused from config; ensures same-origin access.

## 2. Implementation Order
1. **Update iframe Source URL**: Change from absolute `http://localhost:3003/api/resource` to relative `/api/resource`.
2. **Update sandbox Attribute**: Add `allow-popups` and `allow-popups-to-escape-sandbox`.
3. **Implement Click Delegation/Modifier**:
   - Access the same-origin iframe document on load.
   - Attach a click listener to the iframe's document to intercept anchor clicks.
   - For all links that do not target internal hashes (i.e. do not start with `#`), open them in a new tab/window using `window.open(url, '_blank')`.
   - Leave hash anchors (e.g. `<a href="#section">`) to resolve naturally inside the iframe.

## 3. Risks & Mitigations
- **Risk**: Dynamic content inside HTML page (e.g., links generated by scripts after the page loads).
  - *Mitigation*: Use click delegation on the iframe document's body/root instead of a one-time loop over existing `<a>` tags during the `load` event.
- **Risk**: Cross-origin blocked access if proxying is not working.
  - *Mitigation*: Confirm that Vite proxy is correctly configured (it is already configured in `vite.config.js` for `/api`).

## 4. Parallel vs Sequential Work
- The changes are small and self-contained within `DocViewer.jsx`. They must be implemented sequentially within this file.

## 5. Verification Checkpoints
- **Phase 1 Verification**: Verify iframe loads HTML content correctly via the relative `/api/resource...` URL.
- **Phase 2 Verification**: Check browser console log for any sandbox policy violations.
- **Phase 3 Verification**: Click external and internal links in HTML lessons and verify they open in new tabs or navigate correctly.
